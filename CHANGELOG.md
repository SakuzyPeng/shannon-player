### 变更
- 专辑封面与中大型表面容器改用连续圆角（superellipse）：保留各组件原有圆角尺度，令直线与弧线更平滑地衔接；不支持该 CSS 特性的环境自动回退为原有圆角。
- 滚动重写为自定义引擎，目标全平台一致的 macOS 式手感（不再依赖系统原生滚动）：触控板 1:1 跟手；离散滚轮每格转为速度冲量、指数摩擦惯性滑行（总位移与原生等距）；惯性冲到边缘时动量自动转化为橡皮筋过冲，近临界弹簧带初速弹回——修复设计稿方案「贴边硬停、回弹只认贴边后继续滚动」的缺陷。
- 修复触控板快速甩动到极限、回弹前的「抽动」：饱和锁定时清零向内速度——甩动的到达速度已把位置送到极限、使命完成，若保留会让弹簧接管后再向内冲一帧。弹簧从极限点静止起弹，回弹单调。
- 修复触控板快速甩动冲边后「顿一下才回弹」：改为「饱和即回弹」——橡皮筋压缩到极限立即开始弹回，属于同一手势的后续输入（macOS 松手后的系统动量尾巴）一律忽略。尾巴与新手势用两个互补信号区分：事件间隙断开（抬手再滑，>80ms）或幅度跳升（连续无缝再甩，尾巴单调衰减故超 1.5× 即为新滑动）或反向输入即解锁交还控制——连续同向滑动不受限。无速度阈值启发式，跟手拉伸不受影响。
- 衬线字体（Lora / Noto Serif SC 可变字重）改为随应用本地打包，移除对 Google Fonts CDN 的运行时依赖，离线可用。
- 专辑详情页、歌手详情页、歌曲页、歌词页改为按需加载，AMLL 渲染依赖不再进入首屏主包。
- 依据设计稿核对并对齐曲库主界面细节：播放条爱心常驻强调色（实心 = 已收藏，描边 = 未收藏）、进度与音量槽圆角、滚动条圆角、图标栏按压时长与标识边距、封面 hover 罩层时长。
- 补齐设计 Token `--pbk`（深墨播放条底，供歌词页等场景使用）。

### 修复
- 专辑封面播放键的 aria-label 由硬编码中文改为 i18n 插值文案。
- 专辑卡片 hover：去掉 scale 缩放（非整数缩放会重采样已光栅化的圆角、导致边缘毛糙），仅保留抬升；hover 时抬高卡片 z-index，使升起的卡片与投影不再被相邻卡片遮挡（曲库网格与歌手页专辑横排同修）。
