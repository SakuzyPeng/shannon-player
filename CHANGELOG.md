### 新增
- 设计参考资料：整理「香农播放器设计简报」离线导出包到 `docs/design/`。
- Tauri 2 + React 19 + TypeScript + Vite 工程骨架。
- 设计 Token 层：Tailwind v4 + CSS 变量，浅色 / 深色两套语义配色。
- 曲库主界面（「杏色 · 明快 2a」设计）：
  - 图标栏：导航、语言菜单、浅色 / 深色 / 跟随系统三态外观切换。
  - 专辑网格视图（封面 hover 浮现播放键）与详情列表视图，可切换。
  - 悬浮播放条：播放 / 暂停、上一首 / 下一首、随机、循环、进度、音量、收藏。
  - 自绘 macOS 交通灯，接入真实窗口控制（关闭 / 最小化 / 最大化）。
  - 弹性滚动物理与自绘滚动条。
  - Radix UI 右键菜单（键盘导航）与语言下拉菜单。
- 强类型播放器领域模型：播放队列、播放进度、音频设备、循环模式（`src/types/player.ts`）。
- 类型安全的国际化：当前提供简体中文与 English，界面文案实时切换（i18n 架构可扩展更多语言）。

- 无歌词曲目显示空态（在线查找 / 导入 .ttml / .lrc 占位入口，来源注记 AMLL TTML DB）。
- 歌词种子：虚构歌曲「午夜环线」原创逐字歌词（文本来自设计稿）；领域模型 LyricLine 增加逐字时间轴。
- 模拟播放时钟：播放中进度实时推进，曲尾按循环模式自动切曲 / 停止（真实音频引擎接入前的占位）。
- 入口：播放条左侧曲目区点击、曲目右键菜单「查看歌词」。

### 新增（收藏页）
- 收藏页（依据设计稿逐项复刻）：标题 + 收藏统计副标题、分段切换、排序菜单（仅歌曲：最近收藏 / 按标题 / 按歌手）、过滤圆钮、吸顶栏（滚动弹入，含分段与过滤）。
- 分段直接映射播放器收藏状态，点击行内 / 卡片爱心即时取消收藏并从列表消失；各分段独立空态与过滤无结果态。
  - 歌曲：曲库收藏曲目表（当前行均衡器、命中高亮），按歌手排序时切换为「歌手 → 专辑小节 → 组内编号」分组视图。
  - 专辑：复用曲库卡语言，hover 浮现取消收藏 + 播放，标题旁已收藏角标，点击进入专辑页。
  - 歌手：圆形头像（取最新专辑封面），hover 浮现取消收藏，名旁收藏角标，点击进入歌手页。
- 设计稿为三分段（歌曲 / 专辑 / 歌手）；因歌单为可收藏实体、需要归处，此处扩展「歌单」为第四分段（收藏的歌单卡片网格），作为歌单页的正式入口，替换先前临时卡片网格。
- 种子补充：收藏歌手（白鲸电台 / 万能青年旅店 / 陈绮贞 / Radiohead）以对齐设计稿演示态。

### 新增（搜索页）
- 搜索页（依据设计稿逐项复刻）：居中搜索框（聚焦强调圈 + 清空按钮）、范围筛选（全部 / 歌曲 / 专辑 / 歌手 / 歌单）、结果按分组呈现、匹配文字琥珀高亮。
- 真实检索本地曲库：歌曲（标题 / 歌手命中，缩略图 + 当前播放行均衡器）、专辑（卡片）、歌手（圆形头像）、歌单（列表行）；空查询显示最近搜索，无结果有空态。
- 结果点击直达：歌曲替换队列播放，专辑 / 歌手 / 歌单进入对应详情页。
- 入口：图标栏新增「搜索」项（对齐搜索页设计稿的图标栏），⌘F / Ctrl+F 全局唤起，曲库主界面搜索框亦接线跳转；NavKey 增加 search。
- 最近搜索存于 UI store（会话内，回车提交时前插去重、上限 8 条），后期由后端持久化。

### 新增（设置页）

### 新增（首次启动页）
- 首次启动引导（依据设计稿逐项复刻）三态流程：欢迎（空曲库）→ 扫描（旋转唱片 + 进度条 + 实时曲数/专辑数 + 当前文件路径，可后台继续 / 取消）→ 完成摘要（专辑拼贴 + 统计 + 开始听歌）。
- 引导期间覆盖主内容区、隐藏播放条，图标栏保持可用（对齐「全量扫描异步执行」）；「开始听歌」结束引导并回到专辑页。
- 入口：由设置页「添加文件夹…」唤起（onboardingOpen 状态，默认关）；后期由后端根据曲库是否已配置，在真实首次启动时自动触发。
- 扫描进度为占位模拟时钟（180ms 递进），真实扫描进度接入后端后回灌。

### 变更
- 播放条「加入歌单」图标由设计稿自造隐喻（上箭头 + 圆 + 底线，可读性差）改为通用的「列表 + 加号」，与相邻队列图标（三横线）视觉同基因，无需悬停提示即可读出语义。
- 播放条与歌词页的音量控制改为共享原生滑杆：保留 4px 细轨道视觉，同时扩大到 24px 高命中区，交互时浮现竖向圆角矩形手柄，支持连续拖动与方向键微调；音量图标可单击静音并保留原音量，再次单击恢复，播放条有效长度同步增加。
- 播放控制簇补齐语义动效：随机路线交错重组，上一首 / 下一首沿跳轨方向压缩位移，循环状态以半周旋转衔接并在单曲循环时显示动态 `1` 标识；播放条与歌词页共用同一套固定占位控件。
- 播放操作视觉升级：主播放键增加克制的哑光内高光、边界与按压深度；播放 / 暂停符号改为光学居中的圆角几何，并通过同拓扑 SVG 路径实现连续形变，统一覆盖播放条、歌词页、详情页、吸顶栏与封面播放入口。
- 动效系统完成 P1/P2/P3 收口：Radix 菜单与歌词队列补齐退场，歌词栏同步展开收起，收藏 / 播放 / 序号 / 均衡器 / 拖拽柄统一交叉淡化或弹簧反馈；收藏移除、卡片筛选重排、首启阶段、空态与搜索分组平滑过渡，专辑网格封面可共享元素过渡至详情大封面，播放进度按 100ms tick 平滑衔接。为避免 200+ 行曲库的布局计算开销，长歌曲列表筛选仍保持即时更新，仅收藏移除行启用补位动画。
- 窗口自适应：专辑网格（曲库主界面与收藏页专辑分段）由固定 4 列改为 auto-fill 随宽度增减列数（1180 设计稿尺寸仍为 4 列，980 下限 3 列，1920 约 7 列），超宽窗口下封面不再被撑到失衡；歌词页封面尺寸随视口高度收敛（最矮 640 时约 230px），左栏音量条不再贴底。全页面在 [980×640, ∞) 区间实测零横向溢出，布局戒律记入 CLAUDE.md / AGENTS.md。
- 歌曲页按歌手排序时，歌手名称与分组统计会随滚动吸顶，并停靠在歌曲页主吸顶栏下方；下一位歌手进入时自然替换。
- 歌曲页、收藏页与专辑列表加入内缩、低对比度的行间细分隔线：平铺和按歌手分组视图均从标题列开始，保留既有 hover 与当前播放强调；专辑列表同步改为无框表格语言。
- 主内容页切换加入轻量横向淡入淡出过渡：侧栏导航、专辑 / 歌手 / 歌单详情、首次启动引导与全屏歌词页均使用同一机制，侧栏与播放条保持稳定。
- 专辑详情页补齐与歌手、歌单一致的滚动吸顶栏：保留缩略封面、专辑与歌手信息、收藏标识及播放 / 暂停，不重复放入筛选或次级操作。
- 专辑主界面的标题栏改为随内容滚动，并在滚过后以精简吸顶栏保留专辑标识、网格 / 列表切换、排序与全局搜索操作。
- 歌手主界面同样改为标题随内容滚动，并以吸顶栏保留歌手标识、排序与页内过滤。
- 歌曲页吸顶栏补齐排序菜单，与标题栏共用排序状态；过滤展开时会推开排序控件。
- 白色底座式分段选择统一为弹簧滑移动画：专辑网格 / 列表、收藏分类及外观主题切换均按选项实际宽度平滑过渡；专辑视图与收藏分类内容同步淡入淡出，且均遵从系统的减少动态效果偏好。
- 专辑页新增最近添加 / 标题 / 歌手排序，网格与列表视图共享顺序；歌手页新增名称 / 专辑数排序。
- 收藏页排序扩展到全部分段：专辑可按标题或歌手排序，歌手与歌单可按标题排序；切换分段时自动保留或恢复适用的排序项。
- 专辑封面与中大型表面容器改用连续圆角（superellipse）：保留各组件原有圆角尺度，令直线与弧线更平滑地衔接；不支持该 CSS 特性的环境自动回退为原有圆角。
- 滚动：手感交还各平台原生（macOS 触控板橡皮筋、Windows/Linux 滚轮惯性各自沿用系统实现，最贴合本机肌肉记忆），只统一视觉——隐藏系统滚动条、改绘跨平台一致的自绘 thumb（按原生滚动位置直接映射，静止 0.9s 后淡出）。曾评估过自积分速度 + 橡皮筋的自定义引擎，因难以在各平台/输入设备上都还原原生手感，权衡后回退。
- 衬线字体（Lora / Noto Serif SC 可变字重）改为随应用本地打包，移除对 Google Fonts CDN 的运行时依赖，离线可用。
- 专辑详情页、歌手详情页、歌曲页、歌词页改为按需加载，AMLL 渲染依赖不再进入首屏主包。
- 依据设计稿核对并对齐曲库主界面细节：播放条爱心常驻强调色（实心 = 已收藏，描边 = 未收藏）、进度与音量槽圆角、滚动条圆角、图标栏按压时长与标识边距、封面 hover 罩层时长。
- 补齐设计 Token `--pbk`（深墨播放条底，供歌词页等场景使用）。
